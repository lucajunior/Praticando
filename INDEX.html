Questionário

Questão 1: Explique qual é a função de um servidor web em um ambiente de hospedagem e cite pelo menos um exemplo.

Resposta: A função de um servidor web em um ambiente de hospedagem é receber, processar e responder às requisições dos usuários feitas por meio de navegadores, entregando páginas, arquivos ou dados solicitados. Ele atua como intermediário entre o usuário e a aplicação hospedada.
Questão 2: Qual a principal característica do NGINX e por que ele é muito utilizado em projetos de alto desempenho?

Resposta: A principal característica do NGINX é sua alta performance no gerenciamento de conexões simultâneas, graças ao seu modelo de processamento assíncrono e baseado em eventos. Ele é muito utilizado em projetos de alto desempenho porque consome poucos recursos, é escalável e eficiente para servir conteúdo estático, atuar como proxy reverso e balanceador de carga, suportando grandes volumes de tráfego com estabilidade e rapidez.
Questão 3: Diferencie os protocolos HTTP e HTTPS. Por que o HTTPS é recomendado para aplicações modernas?

Resposta:HTTP e HTTPS são protocolos usados para a comunicação entre navegadores e servidores web. A principal diferença é que o HTTPS utiliza criptografia (via SSL/TLS) para proteger os dados transmitidos, enquanto o HTTP envia as informações em texto simples, sem segurança. O HTTPS é recomendado para aplicações modernas porque garante a confidencialidade, integridade e autenticidade dos dados, protegendo usuários contra ataques como interceptação e roubo de informações sensíveis.
Questão 4: Qual é a função das portas em uma rede de computadores e qual a importância da porta 443?

Resposta:As portas em uma rede de computadores funcionam como canais lógicos que identificam serviços ou aplicações específicas em um dispositivo, permitindo que os dados sejam enviados ao destino correto dentro do sistema. A porta 443 é especialmente importante por ser usada para comunicações HTTPS, que garantem conexões seguras e criptografadas na internet, protegendo dados sensíveis como senhas e informações bancárias.
Questão 5: Defina o conceito de endereço IP e explique sua importância em redes de computadores.

Resposta:Endereço IP é um número único que identifica um dispositivo em uma rede, funcionando como seu "endereço" para envio e recebimento de dados. Ele é essencial para garantir que as informações trafeguem corretamente entre os dispositivos em redes locais ou na internet, permitindo a comunicação entre computadores, servidores e outros equipamentos.
Questão 6: Explique o que é hospedagem VPS e em quais situações ela é mais indicada.

Resposta:Hospedagem VPS é um tipo de servidor virtual com recursos dedicados, mesmo estando em um ambiente compartilhado. É mais indicada para sites ou sistemas de médio porte que precisam de mais desempenho, segurança e controle do que a hospedagem compartilhada oferece, como e-commerces, aplicações web e projetos em crescimento.
Questão 7: Descreva as vantagens do servidor web Caddy em relação a outros servidores tradicionais.

Resposta:O servidor web Caddy se destaca por sua configuração simples, segurança integrada e automação de certificados SSL/TLS. Diferente de servidores tradicionais como Apache ou NGINX, ele renova automaticamente certificados HTTPS gratuitos com Let’s Encrypt, exigindo menos configuração manual.
Questão 8: Qual servidor de aplicação é mais indicado para aplicações desenvolvidas na plataforma .NET (C#)? Explique o motivo.

Resposta:O servidor de aplicação mais indicado para aplicações .NET (C#) é o IIS, por ser a solução nativa da Microsoft, com suporte robusto a ASP.NET Core e .NET Framework, além de integração perfeita com o ecossistema Windows.
Questão 9: Descreva o funcionamento do modelo Cliente-Servidor no contexto de sistemas web

Resposta:No modelo Cliente-Servidor em sistemas web, o cliente (geralmente um navegador) envia requisições a um servidor, que processa essas solicitações e retorna uma resposta, como uma página HTML ou dados em JSON. O servidor hospeda os recursos e a lógica da aplicação, enquanto o cliente exibe a interface e interage com o usuário. Esse modelo permite a comunicação estruturada e centralizada, onde múltiplos clientes acessam os serviços fornecidos por um ou mais servidores.
Questão 10: Explique o que é o Apache Tomcat e para qual tipo de aplicação ele é mais indicado.

Resposta:É um servidor web e contêiner de servlets open-source projetado para executar aplicações Java, especialmente Java Servlets, JSP (JavaServer Pages) e Java EE (em versões mais antigas). Ele é leve, eficiente e ideal para aplicações web baseadas em Java
Questão 11: Diferencie, com suas palavras, um servidor web de um servidor de aplicação, destacando suas funções principais

Resposta: Um servidor web é responsável por receber requisições HTTP e entregar conteúdos estáticos, como páginas HTML, imagens e arquivos CSS ou JavaScript. Já um servidor de aplicação executa a lógica de negócio da aplicação, processando dados, acessando bancos de dados e gerando conteúdos dinâmicos, como páginas personalizadas ou respostas em JSON.
Questão 12: Defina o conceito de domínio na internet e sua relação com os endereços IP

Resposta: Um domínio na internet é um nome legível que substitui um endereço IP numérico, facilitando o acesso a sites e serviços sem a necessidade de memorizar números.
Questão 13: Quais as principais vantagens da hospedagem em nuvem em comparação com a hospedagem local?

Resposta: A hospedagem em nuvem oferece escalabilidade flexível (aumento ou redução de recursos sob demanda), alta disponibilidade (com redundância e balanceamento de carga), custo variável (pago conforme o uso, sem investimento inicial em hardware) e acesso remoto global, enquanto a hospedagem local exige infraestrutura física própria, manutenção cara, capacidade fixa e maior risco de downtime, sendo mais indicada apenas para casos com requisitos específicos de segurança ou regulatórios.
Questão 14: Cite dois exemplos de servidores de aplicação e as linguagens/plataformas que eles suportam.

Resposta: Dois exemplos de servidores de aplicação são o Apache Tomcat, especializado em aplicações Java (Servlets/JSP), e o Microsoft IIS, voltado para ASP.NET Core e tecnologias Microsoft, como PHP com configuração adicional.
Questão 15: Por que é essencial a utilização de HTTPS em sites e sistemas atualmente? Quais riscos o HTTP apresenta?

Resposta: O HTTPS é essencial hoje porque criptografa a comunicação entre usuários e servidores, protegendo dados sensíveis (como senhas e informações financeiras) contra interceptação, adulteração e ataques man-in-the-middle. Já o HTTP é inseguro, pois transmite dados em texto puro, facilitando espionagem, roubo de informações e injeção de códigos maliciosos, além de prejudicar a confiança dos usuários e o ranking em motores de busca.
Questão 16: Dê um exemplo de situação onde o uso de hospedagem VPS seria mais vantajoso do que uma hospedagem compartilhada.

Resposta: Um VPS (Virtual Private Server) é mais vantajoso que a hospedagem compartilhada quando o projeto exige recursos dedicados (como CPU, RAM e armazenamento) e maior controle do ambiente — por exemplo, em um ecommerce com tráfego médio/alto, onde a hospedagem compartilhada pode limitar desempenho e segurança, enquanto o VPS oferece escalabilidade, personalização de configurações e isolamento de outros usuários, evitando quedas por sobrecarga alheia.
Questão 17: Diferencie os conceitos de IP e porta em uma rede, explicando suas funções.

Resposta: O endereço IP identifica um dispositivo na rede (como um "endereço residencial"), permitindo que dados sejam roteados até ele, enquanto a porta (um número de 0 a 65535) especifica qual aplicação ou serviço no dispositivo deve receber os dados (como um "número de apartamento")
Questão 18: Qual a importância de um servidor web conseguir lidar com conexões via HTTPS?

Resposta: A capacidade de um servidor web lidar com HTTPS é crucial porque garante a criptografia dos dados trafegados, protegendo informações sensíveis (como logins e pagamentos) contra interceptação e ataques man-in-the-middle. Além disso, o HTTPS aumenta a confiança dos usuários, melhora o ranking em mecanismos de busca (SEO) e atende a requisitos de compliance (como LGPD e PCI DSS), sendo indispensável para segurança e credibilidade na web moderna.
Questão 19: Cite dois exemplos de servidores web e explique suas principais características.

Resposta: Dois exemplos de servidores web são o Apache e o Nginx. O Apache é altamente flexível, com suporte a módulos dinâmicos e ampla compatibilidade com linguagens como PHP, sendo ideal para hospedagem compartilhada e aplicações tradicionais. O Nginx, por sua vez, destaca-se por seu desempenho escalável e baixo consumo de recursos, sendo preferido para sites de alto tráfego, load balancing e conteúdo estático, além de atuar como proxy reverso eficiente. Ambos suportam HTTPS e são open-source.
Questão 20: Defina o que é uma hospedagem compartilhada e em quais casos ela costuma ser utilizada.

Resposta: A hospedagem compartilhada é um serviço onde múltiplos sites utilizam os mesmos recursos de um servidor (como CPU, RAM e armazenamento), tornando-a econômica e simples de gerenciar, porém com desempenho limitado. É ideal para sites pequenos, blogs ou portfólios com tráfego baixo, que não exigem alta performance ou configurações personalizadas, sendo desvantajosa para projetos com alto crescimento ou necessidades específicas de segurança e escalabilidade.
Questão 21: Explique a função de um servidor de aplicação e a diferença em relação a servidores que apenas servem arquivos estáticos.

Resposta: Executa a lógica do sistema, processa dados, conecta com bancos e gera respostas dinâmicas para o usuário. Diferença: Servidores de arquivos estáticos só entregam arquivos prontos (HTML, imagens), enquanto servidores de aplicação criam conteúdo personalizado e interativo conforme a solicitação
Questão 22: Cite um servidor de aplicação usado para projetos em Python e descreva como ele funciona.

Resposta: Ele roda a aplicação Python (geralmente Flask ou Django) como processos separados, recebendo requisições HTTP, processando a lógica da aplicação e enviando respostas para o cliente, garantindo desempenho e escalabilidade
Questão 23: O que é o WildFly (ex-JBoss) e para qual tecnologia ele é indicado

Resposta: O WildFly é um servidor de aplicação Java open-source. Ele é indicado para aplicações desenvolvidas em Java EE/Jakarta EE, oferecendo suporte a componentes como Servlets, EJBs, JMS e outros padrões Java corporativos.
Questão 24: Qual a função da porta 21 em redes de computadores e para qual serviço ela é utilizada?

Resposta: A porta 21 é usada para o serviço FTP (File Transfer Protocol). Função: Ela estabelece a conexão de controle para transferir arquivos entre computadores pela rede.
Questão 25: Explique por que o servidor LiteSpeed é compatível com o Apache e quais as vantagens dessa compatibilidade.

Resposta: O LiteSpeed é compatível com o Apache porque suporta os mesmos arquivos de configuração (como .htaccess) e módulos comuns do Apache. Vantagens: • Facilita a migração de sites do Apache para LiteSpeed sem mudanças no código. • Permite usar regras e configurações já existentes, economizando tempo. • Oferece melhor desempenho e menor uso de recursos que o Apache, mantendo compatibilidade
Questão 26: O que é o Microsoft IIS e em quais situações ele costuma ser adotado?

Resposta: O Microsoft IIS (Internet Information Services) é um servidor web da Microsoft para sistemas Windows. Uso: É adotado principalmente em ambientes que usam tecnologias Microsoft, como ASP.NET, para hospedar sites e aplicações integradas ao ecossistema Windows, especialmente em empresas que usam servidores Windows Server
Questão 27: Explique para que serve o Node.js e em quais tipos de aplicações ele é mais utilizado.

Resposta: O Node.js é uma plataforma que permite executar código JavaScript no servidor. Para que serve: Criar aplicações web rápidas e escaláveis, especialmente que precisam lidar com muitas conexões ao mesmo tempo. Usos comuns: APIs, servidores em tempo real (chat, jogos), aplicações com I/O intensivo (arquivos, bancos de dados).
Questão 28: Descreva a função do servidor GlassFish e para qual linguagem ele é destinado

Resposta: O GlassFish é um servidor de aplicação que executa aplicações baseadas em Java EE/Jakarta EE. Função: Ele gerencia a execução de componentes Java, como Servlets, EJBs e Web Services, facilitando o desenvolvimento e a implantação de sistemas corporativos em Java.
Questão 29: O que são Puma e Unicorn e em quais projetos eles são normalmente utilizados?

Resposta: Puma e Unicorn são servidores de aplicação para Ruby, usados principalmente em projetos Rails. • Puma: Suporta múltiplas threads e conexões simultâneas, ideal para aplicações que precisam de alta concorrência. • Unicorn: Trabalha com múltiplos processos, melhor para aplicações que exigem estabilidade e isolamento. Ambos são usados para rodar apps web Ruby em produção
Questão 30: Qual a importância da escolha adequada entre hospedagem local, compartilhada, VPS e nuvem no planejamento de um projeto web?

Resposta: A escolha adequada impacta diretamente em: • Desempenho: Recursos disponíveis afetam velocidade e estabilidade. • Escalabilidade: Capacidade de crescer conforme o tráfego aumenta. • Segurança: Diferentes níveis de proteção e controle. • Custo: Cada tipo tem preço e manutenção diferentes. • Controle: VPS e nuvem oferecem mais liberdade para configurar
